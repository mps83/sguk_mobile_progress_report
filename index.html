1. Executive Summary

This document details the engineering progress of the SGUK Guard mobile application. The project has successfully transitioned from the architectural planning phase to a functional Alpha release. The core foundation is built upon a scalable, type-safe React Native environment, ensuring cross-platform compatibility (iOS & Android) and high performance. Key modules including Authentication, Job Management, and Real-time Messaging have been implemented and partially integrated with the Supabase backend.

2. System Architecture

The application utilizes a modern "Serverless" architecture, leveraging Supabase for backend services and Expo for the frontend runtime. This ensures low maintenance overhead and high scalability.

2.1. Frontend Technology Stack

  • Core Framework: React Native 0.76 with Expo SDK 52.
  • Language: TypeScript 5.3 (Strict Mode enabled for maximum type safety).
  • Routing: expo-router v4 (File-based routing with deep linking support).
  • State Management: React Context API for global session state; Local state for UI components.
  • Styling: Custom Design System utilizing StyleSheet API with theme-aware tokens.

2.2. Backend & Infrastructure

  • Database: PostgreSQL 15 (via Supabase).
  • Authentication: Supabase Auth (JWT-based secure sessions).
  • Real-time: Supabase Realtime (WebSockets) for chat and job updates.
  • Edge Functions: Deno-based serverless functions for Push Notifications.

3. Detailed Feature Implementation

3.1. Authentication Module Complete

A robust authentication flow has been engineered to ensure secure access control.

  • Login Interface: Implemented in app/login.tsx with form validation and error feedback.
  • Session Persistence: Utilizes @react-native-async-storage/async-storage to cache JWT tokens, allowing users to remain logged in across app restarts.
  • Security: All API requests are authenticated via RLS (Row Level Security) policies on the database level.

3.2. Guard Dashboard & Job Management Complete

The dashboard serves as the operational center for guards, aggregating data from multiple streams.

  • Dashboard Grid: A responsive grid layout (DashboardGrid.tsx) that adapts to different screen sizes.
  • Active Job Tracking: The ActiveJobCard component subscribes to the shifts table to display the current ongoing shift with a "Live" status indicator.
  • Schedule Management: UpcomingJobCard renders a sorted list of future shifts, calculating time-until-start dynamically.
  • Job Discovery: The JobListingCard allows guards to browse available shifts. (Accept/Decline API integration in final testing).

3.3. Secure Communication System In Progress

A dedicated messaging module designed for operational security and team coordination.

  • UI Implementation: A high-performance Chat UI (components/chat/) capable of rendering thousands of messages efficiently using FlatList optimization.
  • UX Features: Includes keyboard handling (KeyboardAvoidingView), message timestamp grouping, and visual distinction between sender/receiver.
  • Integration: Currently connecting to the messages table for real-time WebSocket updates.

3.4. Notification Services Complete

A comprehensive push notification infrastructure to keep guards informed of critical updates.

  • Token Management: The usePushNotifications hook handles the complex logic of requesting OS-level permissions and retrieving the Expo Push Token.
  • Server-Side Dispatch: A dedicated Edge Function (supabase/functions/push-notification) has been written to securely dispatch notifications via FCM (Android) and APNs (iOS).
